# Advantages of modules

1. Allows one to split the code into multiple files
2. Allows code reuse across different files.
3. Encapsulation - keeping internal details hidden and only providing a single interface for the public to use.
4. Managing dependencies - A good module system should make it easy for
   module developers to build on top of existing modules,

N/B
IIFE (Immediately Invoked Function Expressions) - self invoking functions

# ESM vs CommonJS

Main difference is that ESM modules are static, meaning imports are described at the top level of every module outside a control flow.
