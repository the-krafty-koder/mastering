# Semantic elements

Elements with meaning.

<section> -> defines a section in a document.
<article> -> specifies independent, self contained content. Examples of usage: forum posts, blog posts, user comments.
<header> -> represents a container for introductionary content or navigational links.
<footer> -> defines a footer for a document.
<nav> -> defines a set of navigation links.
<aside> -> defines some content aside from the content it is placed in, eg in a sidebar.
<figure> -> specifies self-contained content, like illustrations, photos etc.
<main> -> specifies main content of a document.

# Why use them

1. Accessibility -> help screen readers understand the page structure.
2. SEO - search engines use them to better understand page content , thus improving rankings.
3. Standardization - future-proofs software by building more robust apps.

# HTML 5

- Not a part of semantic html topic.
- Added new elements that help structure a webpage better and make it more interactive eg <article>, <aside>, <header>, <footer>, <main>, <nav>, <section>

Features of HTML5

1. Semantic elements
2. Multimedia support - native support of video and audio tags.
3. Canvas API- drawing animations directly in the browser.
4. Geolocation API - access to the users geographical location.
5. Web storage - localStorage and sessionStorage
6. Form enhancements - new input types (email, number, date) and form validation make it easier to validate forms.
7. Drag and drop API - allows dnd , enhancing interactivity.
8. Websocket API - enables bidirectional communication.

Deprecated elements
<big>, <basefont>, <acronym>, <font>, <strike>

# Advantages of HTML5

1. Better perfomance - faster loading times with fewer dependencies.
2. Cross-platform - works across all major browsers and mobile devices.
3. Offline capability - works offline with web storage and app cache.
4. Geolocation
5. Simplified code with semantic html tags
6. Mobile friendly - optimized for a smoother mobile experience.

# Disadvantages og HTML5

1. inconsistencies - older browsers/devices may not support HTML5
2. Complexity -> features like Websockets and APIs can be difficult for beginners to understand.
3. Security Risks -> New features like Web Storage can potentially expose user data if not handled properly.
